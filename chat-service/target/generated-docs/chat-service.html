1. 회원관련 API

1.1. 회원가입 API

회원가입 기능을 제공하는 API이지만, 회원가입은 채팅 서비스에서 진행하지 않고, 메인 서버에서 진행합니다.

따라서 임시 회원을 만들고 테스트 하는 용도로 사용합니다.

Request 예시입니다.

POST /api/v1/webrtc/chat/register HTTP/1.1
Content-Type: application/json;charset=UTF-8
Accept: application/json
Content-Length: 79
Host: localhost:8080

{
  "nickname" : "nickname1",
  "password" : "password",
  "email" : "email1"
}

1.2. jwt 토큰발급 API

jwt Access token을 발급 받습니다.

  1. 토큰을 발급받을때는 Header에 별도의 처리를 안해도 됩니다.

  2. email과 password로 가입된 사용자인지 확인합니다.

  3. 이후의 모든 Request는 Header Authorization에 토큰을 넣어야 하며 형식은 아래와 같습니다.

  4. "jwt " + "발급받은 토큰" jwt 뒤에 공백이 있음에 주의하세요.

Request 예시입니다.

POST /api/v1/webrtc/chat/authenticate HTTP/1.1
Content-Type: application/json;charset=UTF-8
Accept: application/json
Content-Length: 51
Host: localhost:8080

{
  "email" : "email1",
  "password" : "password"
}

2. 채널관련 API

2.1. 채널생성 API

새로운 채널을 생성합니다. 같은 이름의 채널이 이미 존재하거나, Jwt-Token이 올바르지 않은 경우 채널이 생성되지 않습니다.

  1. header에 jwt access 토큰을 넣어야 합니다.

  2. hashTags는 공백일 경우 최소 빈 배열을 넘겨주세요.

  3. hashTags 개수의 제한은 없으나, UI에 문제가 발생할 수도 있기에 정책의 설정이 필요합니다.

  4. 문자 채팅과 음성 채팅방은 분리가 되어 있습니다. 채널 타입이 추가되는 확장성을 위해 Enum 타입으로 정의합니다. 현재 존재하는 채널 타입의 종류는 아래와 같습니다.

    TEXT, VOIP

Request 예시입니다.

POST /api/v1/webrtc/chat/channel HTTP/1.1
Content-Type: application/json;charset=UTF-8
Authorization: jwt eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJlbWFpbDEiLCJleHAiOjE2NTkwMzA3MDQsImlhdCI6MTY1OTAxMjcwNH0.zovBIdGLT5BxgLZbHJFqQeOygBEfu-OfctkkDE0w9Uj6FCtyIG5pzho5RY3MU09UyGDqb6gOEphMhs5vGyznvw
Accept: application/json
Content-Length: 105
Host: localhost:8080

{
  "channelName" : "channelName1",
  "hashTags" : [ "tag1", "tag2", "tag3" ],
  "channelType" : "TEXT"
}

  2. Jwt Access Token이 만료되었습니다.

  3. 올바르지 않은 Jwt Token 형식입니다.

HTTP/1.1 401 Unauthorized
Vary: Origin
Vary: Access-Control-Request-Method
Vary: Access-Control-Request-Headers

2.3. 전체 채널목록들 반환 API

채널 목록들을 반환합니다. 채널 목록을 불러오는 API는 현재 입장하고 있는 채널 목록을 불러오는 API와 전체 채널 목록들을 불러오는 API가 존재합니다. 해당 API는 전체 채널들을 불러오는 API 입니다.

요청을 보낼때 고려해야 하는 것은 크게 3가지입니다.

  1. Jwt Access 토큰을 Header에 넣어야 합니다.

  2. 몇번째 채널 목록을 불러올 것인지 index 값

  3. 채널 목록의 정렬 조건

채널 목록에 대한 요청은 스크롤 페이징을 적용했습니다. 한번에 모든 채널 목록을 불러오지 않고 페이징을 적용한 이유는 아래와 같으며, 한번에 최대 '20’개의 채널 목록을 반환합니다.

  1. 한번에 모든 채널을 불러오는 요청을 처리함으로써 request 처리 비용이 증가합니다. (ex : 서버 메모리 부하, DB 부하, 네트워크 부하 증가로 response 받는 시간 증가 등 )

  2. 채널에는 TTL이 존재합니다. TTL은 DB에 따로 저장하지 않으며 채널 목록을 불러온 후 response를 보내기 직전 처리합니다. 따라서 불러온 채널 목록의 개수에 따라 처리 시간이 증가합니다.

  3. 사용자의 가독성이 떨어집니다.

따라서 request를 보낼때 몇번째 페이지에 대한 요청인지 {idx} 값을 넣어서 보내주셔야 합니다.

ex) 첫 채널 목록을 불러올때 : idx = 0, 두번째 목록을 불러올 때 : idx = 1

채널 목록의 정렬 조건은 어떤 기준으로 채널 정보들을 불러올지 결정합니다. 따라서 요청을 보낼때 url에 이 기준을 넣어서 보내주셔야 합니다. 현재 정렬 조건은 아래와 같은 것들이 존재합니다.

partiASC, partiDESC

2.4. 나의 채널목록들 반환 API

내가 입장한 채널 목록들을 반환합니다. 채널 목록을 불러오는 API는 현재 입장하고 있는 채널 목록을 불러오는 API와 전체 채널 목록들을 불러오는 API가 존재합니다. 해당 API는 사용자가 입장한 모든 채널들을 불러오는 API 입니다.

요청을 보낼때 고려해야 하는 것은 크게 3가지가 존재합니다.

  1. Jwt Access 토큰을 Header에 넣어야 합니다.

  2. 몇번째 채널 목록을 불러올 것인지 index 값

  3. 채널 목록의 정렬 조건

채널 목록에 대한 요청은 스크롤 페이징을 적용했습니다. 한번에 모든 채널 목록을 불러오지 않고 페이징을 적용한 이유는 아래와 같으며, 한번에 최대 20개의 채널 목록을 반환합니다.

  1. 한번에 모든 채널을 불러오는 요청을 처리함으로써 request 처리 비용이 증가합니다. (ex : 서버 메모리 부하, DB 부하, 네트워크 부하 증가로 response 받는 시간 증가 등 )

  2. channel에는 TTL이 존재합니다. TTL은 DB에 따로 저장하지 않으며 채널 목록을 불러온 후 response를 보내기 직전 처리합니다. 따라서 불러온 채널 목록의 양에따라 처리 시간이 증가합니다.

  3. 사용자의 가독성이 떨어집니다.

2.5. 해시태그 채널목록들 반환 API

해시태그로 연관된 채널 목록들을 반환합니다.

요청을 보낼때 고려해야 하는 것은 크게 4가지가 존재합니다.

  1. Jwt Access 토큰을 Header에 넣어야 합니다.

  2. 몇번째 채널 목록을 불러올 것인지 index 값

  3. 채널 목록의 정렬 조건

  4. 찾고자 하는 해시태그값

채널 목록에 대한 요청은 스크롤 페이징을 적용했습니다. 한번에 모든 채널 목록을 불러오지 않고 페이징을 적용한 이유는 아래와 같으며, 한번에 최대 20개의 채널 목록을 반환합니다.

  1. 한번에 모든 채널을 불러오는 요청을 처리함으로써 request 처리 비용이 증가합니다. (ex : 서버 메모리 부하, DB 부하, 네트워크 부하 증가로 response 받는 시간 증가 등 )

  2. channel에는 TTL이 존재합니다. TTL은 DB에 따로 저장하지 않으며 채널 목록을 불러온 후 response를 보내기 직전 처리합니다. 따라서 불러온 채널 목록의 양에따라 처리 시간이 증가합니다.

  3. 사용자의 가독성이 떨어집니다.

따라서 request를 보낼때 몇번째 페이지에 대한 요청인지 {idx} 값을 넣어서 보내주셔야 합니다.

ex) 첫 채널 목록을 불러올때 : idx = 0, 두번째 목록을 불러올 때 : idx = 1

채널 목록의 정렬 조건은 어떤 기준으로 채널 정보들을 불러올지 결정합니다. 따라서 요청을 보낼때 url에 이 기준을 넣어서 보내주셔야 합니다. 현재 정렬 조건은 아래와 같은 것들이 존재합니다.

2.7. 채널 수명 연장 API
